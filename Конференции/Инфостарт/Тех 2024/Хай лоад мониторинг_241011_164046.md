Информация от кластера 1С важна как дополнение для понимания всей картины той или иной ситуации. Но только на основании этой информации вы не докопаете до кода, вызвавшего проблему. 
Нагрузку от оборудования толковые админы так или иначе собирают, обычно в zabbix. Это уже хорошо, просто запросите доступ. Если не собирают, лайфхак такой - чуть что не так, сразу пишите админам, что наверняка причина в сервере. На ответное сообщение "с оборудованием все Ок", спрашивайте "чем докажешь?". 
Самое важное - работа с логами технологического журнала. 
Поверьте, причины большого количества проблем можно найти с помощью логов ТЖ. Работа с логами посредством ElasticSearch (или ClickHouse) через интерфейс Grafana (Kibana) дает просто невероятное удобство и скорость.
