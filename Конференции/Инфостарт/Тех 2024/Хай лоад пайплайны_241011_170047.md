Считать hash по тексту запроса надо для простоты группировки по тексту запросов (чтобы сделать более читаемую таблицу).
Вторая возможность - интегрироваться с трассировками запросов на уровне СУБД (об этом чуть дальше). 
Для расчета hash можно использовать Vector, Logstash не умеет. 
Не хотите долго возиться с правилами парсинга ТЖ - в версии 8.3.25 его можно формировать в формате JSON. По сути вам останется только передать JSON в эластик или клик.
